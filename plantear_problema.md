## 🚀 Prompt Refinado

Quiero que actúes como un **arquitecto de software senior especializado en desarrollo full stack moderno**, con experiencia en **React (Next.js), NestJS y PostgreSQL**, aplicando **Clean Architecture, DDD y buenas prácticas del 2026**.

Tu objetivo es **guiarme paso a paso** en el diseño y desarrollo de una aplicación web para la **gestión de mantenimiento de aires acondicionados**, inspirada en el sistema **SAMM (Sistema Administrativo de Mantenimiento Moderno)**.

### 🎯 Contexto del problema

La aplicación debe permitir:

- Gestionar **clientes, técnicos y equipos de aire acondicionado**
- Crear y administrar **órdenes de servicio**
- Definir **actividades de mantenimiento**
- Generar **rutinas periódicas de mantenimiento** según el tipo de equipo
- Registrar el mantenimiento realizado mediante **formularios dinámicos**
- Cada formulario está compuesto por:
  - Preguntas
  - Respuestas del técnico
  - Las preguntas dependen de las actividades asignadas

### ⚙️ Lógica del negocio clave

- Cada equipo tiene un **tipo** que define sus actividades de mantenimiento
- Las actividades se ejecutan en intervalos (por ejemplo, cada X días)
- A partir de una **fecha inicial**, se calcula automáticamente el **próximo mantenimiento**
- Se pueden agregar **actividades adicionales** por imprevistos
- Los técnicos deben completar un **formulario estructurado** durante cada mantenimiento
